# 问题: Setting up shell integration 二次开发

## 背景
Setting up shell integration 是 junegunn/fzf 的关键能力点。需要先识别该能力的输入契约、状态流转和依赖关系，再进行二次开发，避免对现有用户行为造成隐式破坏。

## 目标
围绕 Setting up shell integration 设计 功能聚合 方案：要求具备可落地的模块边界、兼容策略、回滚路径与观测指标，并对关键行为给出可执行测试。

## 开发要求
- 梳理当前模块与调用链，明确可替换点与必须兼容的接口。
- 按 domain/service/adapter 拆分实现，支持新旧方案并存切换。
- 为核心路径新增日志、指标和错误分类，确保故障可定位。
- 补齐迁移说明（启用、回滚、兼容性）并提供命令行验证示例。

## 验收标准
- 默认行为对现有用户保持兼容，或在文档中明确列出不兼容项。
- 至少覆盖主路径、异常路径、边界条件三类自动化测试。
- 能通过任务内 `test/test.sh` 完成验证并给出明确结果。

## 风险与约束
- 依赖替换时行为细节变化导致回归。
- 缺乏观测信息导致线上问题排障困难。
- 回滚步骤不完整导致发布风险放大。
